# MCP 混合方案实施计划

## 概述

本文档详细描述了 MCP（Model Context Protocol）在 Project Animus 中的实现计划，采用**混合技术路线**：
- **邮箱服务**：使用 IMAP 直接实现（163、QQ、Outlook）
- **本地能力**：预留 MCP 架构，未来引入官方 MCP Server
- **云服务 API**：保持直接调用（天气、搜索等）

**目标版本**：V2.5  
**开始时间**：2025-01-08  
**当前状态**：阶段1已完成（邮箱主动提醒）

---

## 技术路线决策

### 核心原则

**务实优先，渐进式引入 MCP**

经过评估，我们采用混合方案而非纯 MCP 方案，原因：

1. **国内邮箱服务**：163、QQ 邮箱没有现成的 MCP Server，且通过 IMAP 协议直接连接更稳定
2. **现有代码基础**：`email_providers.py` 已实现 IMAP 连接，无需重构
3. **MCP 生态限制**：MCP 主要由国外社区维护，国内服务支持有限
4. **渐进式扩展**：先完善核心功能，再逐步引入 MCP 用于本地能力（filesystem、GitHub 等）

### 实施策略

- **阶段1（已完成）**：邮箱主动提醒 - 使用 IMAP 直接实现
- **阶段2（未来）**：工具系统统一 - 为 MCP 工具预留接口
- **阶段3（未来）**：本地能力引入 - 使用官方 MCP Server（filesystem、GitHub）

## 技术选型

### MCP Client SDK

**推荐方案**：使用 `mcp` Python SDK
- **包名**：`mcp`（注意：`@modelcontextprotocol/sdk-python` 是 npm 格式，Python 只需 `pip install mcp`）
- **安装**：`pip install mcp`
- **文档**：https://modelcontextprotocol.io

**备选方案**：LangChain MCP 集成（如果 LangChain 提供官方支持）

### 连接方式

1. **stdio（本地进程）**：用于本地 MCP Server（如文件系统）- **标准 MCP 连接方式**
2. **SSE（Server-Sent Events）**：用于远程 MCP Server - **标准 MCP 连接方式**
3. **直接 API 调用**：用于云服务（邮箱 IMAP、天气 API 等）- **非 MCP 协议，但更实用**

**注意**：MCP 协议标准只支持 stdio 和 SSE。HTTP 不是 MCP 的原生连接方式，文档中提到的 "HTTP" 实际是指直接调用 REST API。

### 安全机制

1. **OAuth 2.0 授权**：用于第三方服务（Google、Microsoft、Slack 等）
2. **权限白名单**：只允许授权的工具和服务
3. **沙箱环境**：限制文件系统访问范围
4. **隐私保护**：敏感数据（邮件内容、财务信息）需要用户明确授权

### MCP Manager 设计

**文件位置**：`mcp_manager.py`

**核心功能**：
1. 管理多个 MCP Client 连接
2. 工具调用路由和分发
3. 权限管理和安全检查
4. 错误处理和重试机制
5. 主动提醒触发机制

**接口设计**：
```python
class MCPManager:
    def __init__(self):
        self.clients = {}  # {server_name: MCPClient}
        self.tool_registry = {}  # {tool_name: ToolInfo}
    
    def add_server(self, server_name: str, config: dict):
        """添加 MCP Server 连接"""
        pass
    
    def call_tool(self, tool_name: str, args: dict) -> dict:
        """调用 MCP 工具"""
        pass
    
    def get_available_tools(self) -> List[ToolInfo]:
        """获取所有可用工具"""
        pass
    
    def check_permissions(self, tool_name: str) -> bool:
        """检查工具调用权限"""
        pass
```

### 工具调用流程

```
用户输入/定时触发
    ↓
reasoning_node 检测需要工具
    ↓
MCP Manager 查找工具
    ↓
权限检查
    ↓
调用 MCP Server
    ↓
返回结果
    ↓
reasoning_node 生成回复/触发主动提醒
```

## 风险评估与缓解

### 技术风险

- **MCP Server 不稳定**
  - 影响：高
  - 概率：中
  - 缓解措施：实现重试机制、降级方案

- **OAuth 授权失败**
  - 影响：高
  - 概率：低
  - 缓解措施：完善的错误处理、用户引导

- **API 限流**
  - 影响：中
  - 概率：中
  - 缓解措施：实现请求限流、缓存机制

- **网络延迟**
  - 影响：中
  - 概率：中
  - 缓解措施：异步调用、超时设置

- **IMAP 连接不稳定**（已实施）
  - 影响：中
  - 概率：中
  - 缓解措施：已实现自动重连机制、错误处理

### 安全风险

- **Token 泄露**
  - 影响：高
  - 概率：低
  - 缓解措施：加密存储、定期刷新

- **权限滥用**
  - 影响：高
  - 概率：低
  - 缓解措施：权限白名单、最小权限原则

- **数据泄露**
  - 影响：高
  - 概率：低
  - 缓解措施：只读取必要数据、加密传输

- **邮箱授权码泄露**（已实施）
  - 影响：高
  - 概率：低
  - 缓解措施：配置文件中不保存密码，使用环境变量或安全存储

### 用户体验风险

- **提醒过于频繁**
  - 影响：中
  - 概率：中
  - 缓解措施：智能频率控制、用户配置（已实现：记录已提醒邮件，避免重复）

- **提醒时机不当**
  - 影响：中
  - 概率：中
  - 缓解措施：专注模式检测、勿扰时间（已实现）

- **隐私担忧**
  - 影响：高
  - 概率：中
  - 缓解措施：透明化数据使用、用户控制（已实现：只读取元数据，不读取内容）
