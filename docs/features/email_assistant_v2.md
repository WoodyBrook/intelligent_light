# AI-Native 邮件助理详细设计规格 (v2.0)

## 1. 设计初衷

传统的邮件提醒仅是“消息的搬运工”，往往会增加用户的认知负担。Animus 的邮件助理旨在通过 **AI-Native** 的方式，将邮件转化为有温度、有情感、且绝对隐私安全的“猫系转述”。

核心目标：
- **隐私本地化**：邮件正文（Body）不离开本地设备。
- **情感化表达**：以“家猫”的视角转述，而非冷冰冰的通知。
- **智能分流**：根据 AI 评分自动决定提醒强度。

## 2. 核心处理链路

系统采用“本地预处理 + 路由分发”的链路。

```mermaid
graph TD
    A[IMAP Server] -->|5min Polling| B[Email Checker]
    B -->|Raw Email| C[local_email_node (Ollama)]
    
    subgraph Local [本地安全区]
    C --> C1[情感标签提取]
    C --> C2[重要性语义打分]
    C --> C3[隐私脱敏摘要]
    end
    
    C -->|Summary + Score + Meta| D[Animus Graph Router]
    D -->|Owner & High Score| E[Reasoning Node]
    D -->|Owner & Low Score| F[Monologue/Batch]
    D -->|Guest| G[貓系拒绝逻辑]
    
    E -->|情感化包装| H[语音+灯光反馈]
```

## 3. 详细规格定义

### 3.1 隐私与安全防御（核心）

为了防止“提示词注入”泄露隐私，采用双层防御架构：

1.  **第一层：确定性逻辑硬门禁 (Hard Gate)**
    - 在 `local_email_node` 代码层进行判断。
    - 若 `current_user_status != 'Owner'`，代码层面直接阻断对邮件正文的处理，仅返回：`summary="那是主人的秘密，我不能看喵"`。
    - **逻辑拦截优先于模型推理**，确保即使发生提示词注入，模型也拿不到正文数据。

2.  **第二层：本地脱敏规范 (Sanitization)**
    - 本地模型在生成摘要时，必须对以下信息进行正则占位符替换：
        - 验证码/纯数字 -> `[数字]`
        - 银行卡/账号 -> `[账号]`
        - 详细地址 -> `[地点]`

3.  **第三层：模型软拒绝 (Soft Rejection)**
    - 在本地 LLM 的 System Prompt 中明确定义：“作为主人的猫，你绝对忠诚，严禁向任何访客泄露主人的私人信件内容。”

### 3.2 AI 评估维度

本地模型 (Ollama/Llama 3.2) 对每封邮件输出以下结构：

| 维度 | 定义 | 说明 |
| :--- | :--- | :--- |
| **Sentiment** | 情感标签 | `happy`, `anxious`, `urgent`, `social`, `routine` |
| **Importance** | 语义打分 (0-100) | 结合紧急度、利益相关度计算。 |
| **Task** | 任务提取 | 识别邮件中是否包含需要用户操作的任务。 |
| **Summary** | 猫系摘要 | 侧重于“谁、为了什么事、什么心情”的 50 字内概括。 |

### 3.3 动态权重矩阵

系统会结合 `LampState` 中的上下文动态调整 `Importance Score`：

- **记忆相关性**：若邮件内容与 `recent_memories` 中用户提到的“项目”、“人物”匹配，权重 +20。
- **用户状态**：
    - `Focus Mode`：只有 `Score > 90` 的邮件会通过紫色呼吸灯静默提示。
    - `Normal Mode`：`Score > 40` 即发起语音转述。
    - `Sleep Mode`：全部静默，记入独白。

## 4. “猫系”转述话术示例

| 邮件类型 | 传统通知 | Animus 猫系转述 |
| :--- | :--- | :--- |
| **朋友好消息** | 您收到一封张三的邮件：下周来我家玩。 | “（跳上桌面）主人，张三发信说下周有大餐吃喵！他看起来超级开心，我们要去吗？” |
| **老板催进度** | 收到紧急邮件：请在5点前提交报告。 | “（耳朵抖动）那个...老板好像有点急，他在问报告的事喵。要不要先理理他？” |
| **垃圾邮件** | 收到一封促销邮件：全场五折。 | （静默处理，仅记入 Monologue）“今天又是这些没营养的传单喵...” |
| **访客询问** | 权限不足。 | “（歪头）那是主人的悄悄话，我才不会告诉你这个外人喵！” |
