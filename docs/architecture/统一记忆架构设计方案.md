## 一、架构总览

│  │ - 存储：ChromaDB向量库                            │  │

└─────────────────────────────────────────────────────────┘

## 二、各层详细定义

### 2.1 持久化层 (Persistent Layer)

#### 2.1.1 Profile (user_profile.json)

**定义**: 长期稳定的结构化用户画像

**存储位置**: `./data/chroma_db_actions/user_profile.json`

**数据结构**:
```python
class UserProfile(BaseModel):
    version: str = "2.0"
    
    # === 基础信息 ===
    name: Optional[str] = None
    gender: Optional[str] = None  # "male" | "female" | "other"
    age: Optional[int] = None
    occupation: Optional[str] = None
    birthday: Optional[str] = None  # "MM-DD"
    
    # === 地理位置 ===
    home_city: Optional[str] = None  # 常住地（长期稳定）
    timezone: Optional[str] = None  # "Asia/Shanghai"
    
    # === 时间规律（用于主动行为）===
    typical_wake_time: Optional[str] = None  # "07:30"
    typical_sleep_time: Optional[str] = None  # "23:00"
    active_hours: Optional[List[str]] = []  # ["09:00-12:00", "14:00-18:00"]
    
    # === 核心偏好（结构化）===
    core_preferences: List[str] = []  # Top 5核心偏好，如 ["喜欢咖啡", "不喜欢吵闹"]
    
    # 结构化偏好对象
    music_preferences: Optional[Dict[str, Any]] = None  # {liked_artists: [...], liked_genres: [...]}
    news_preferences: Optional[Dict[str, Any]] = None  # {categories: [...], keywords: [...]}
    light_preferences: Optional[Dict[str, Any]] = None  # {preferred_color_temp: 4000, ...}
    
    # === 重要日期 ===
    important_dates: Optional[List[Dict[str, Any]]] = []  # [{"date": "MM-DD", "name": "...", "type": "..."}]
    
    # === 元数据 ===
    last_updated: float = 0.0
```

**特点**:
- ✅ 长期稳定，不频繁变化
- ✅ 结构化数据，字段明确
- ✅ 每次对话都需要，必须默认加载
- ✅ 影响主动行为的关键信息

**更新规则**:
- 只有核心画像信息才更新Profile
- 更新后立即持久化到JSON文件
- 更新后同步到State.user_profile

#### 2.1.2 ROM (ChromaDB)

**定义**: 非结构化的长期记忆

**存储位置**: `./data/chroma_db_actions/user_memory/`

**数据结构**: 向量文档
```python
Document(
    page_content="用户喜欢去公园",  # 文本内容
    metadata={
        "timestamp": 1704067200.0,
        "date": "2024-01-01 10:30:00",
        "category": "preference",  # user_profile | habit | preference | relationship
        "source": "conversation",
        "confidence": 0.8
    }
)
```

**应该包含**:
- ✅ 次要偏好（非核心）："用户偶尔喜欢在雨天听音乐"
- ✅ 历史经历（有长期价值）："用户去年去了日本旅游"
- ✅ 习惯和规律："用户习惯周末去公园"
- ✅ 故事和细节："用户提到过小时候的趣事"
- ✅ 重要人物："用户的朋友叫小明"

**不应该包含**:
- ❌ 一次性事件（会过期）："用户昨天去了公园"
- ❌ 临时信息（会过期）："用户明天要开会"
- ❌ 对话中的临时话题："今天天气真好"

**检索方式**: 按需Tool调用，语义检索

### 2.2 运行时状态层 (Runtime State Layer)

#### 2.2.1 State.user_profile (Dict)

**定义**: Profile的内存镜像

**数据来源**: 从Profile JSON文件加载

**同步规则**:
- 启动时：从Profile加载 → State.user_profile
- 运行时：如果State.user_profile被更新 → 同步回Profile JSON
- 更新Profile时：Profile更新 → 同步到State.user_profile

**用途**: 
- 快速访问（避免频繁读取JSON）
- 运行时状态管理
- 作为memory_context的数据源

#### 2.2.2 State.context_signals (Dict)

**定义**: 当前临时状态（不持久化）

**包含内容**:
```python
{
    "current_time": float,  # 时间戳
    "current_hour": int,  # 当前小时
    "current_day": int,  # 当前星期
    "current_location": Optional[str],  # 临时位置（出差、旅游）
    "activity_level": str,  # "active" | "recent" | "away"
    "focus_mode": bool,
    "current_mood": Optional[str],  # 临时情绪
    "current_task": Optional[str]  # 当前任务
}
```

**特点**:
- ✅ 临时状态，每次启动重新计算
- ✅ 不持久化（会过期）
- ✅ 用于实时上下文

**更新规则**:
- 每次OODA循环更新
- 从传感器、时间等实时计算
- 不保存到Profile或ROM

#### 2.2.3 State.memory_context (Dict)

**定义**: 格式化的上下文，用于注入System Prompt

**结构**:
```python
{
    "user_profile": str,  # Profile格式化后的文本
    "user_memories": List[str],  # ROM检索到的相关记忆（按需）
    "action_patterns": List[str],  # Action Library检索结果
    "realtime_context": str,  # 实时上下文（时间、位置等）
    "search_query": str  # 当前查询
}
```

**生成流程**:
1. 从Profile加载 → 格式化为文本
2. 从ROM检索（按需）→ 添加到user_memories
3. 从context_signals生成 → realtime_context
4. 合并 → memory_context

---

### 2.3 上下文层 (Context Layer)

#### 2.3.1 System Prompt

**定义**: 注入到LLM的上下文

**包含内容**:
- Profile格式化文本（必须）
- ROM检索结果（按需）
- 实时上下文（时间、位置等）
- 对话历史（最近N轮）

**格式**: XML结构化标签

## 三、数据分类规则

### 3.1 应该存在Profile的数据

**标准**: 长期稳定 + 结构化 + 核心画像 + 影响主动行为

**示例**:
- ✅ 姓名、性别、年龄、职业
- ✅ 常住地（home_city）
- ✅ 作息时间（typical_wake_time, typical_sleep_time）
- ✅ 核心偏好（Top 5）："喜欢咖啡", "不喜欢吵闹"
- ✅ 结构化偏好：音乐偏好（艺术家列表）、新闻偏好（类别列表）

### 3.2 应该存在ROM的数据

**标准**: 长期价值 + 非结构化 + 次要信息 + 按需检索

**示例**:
- ✅ 次要偏好："用户偶尔喜欢在雨天听音乐"
- ✅ 历史经历："用户去年去了日本旅游"
- ✅ 习惯规律："用户习惯周末去公园"
- ✅ 故事细节："用户提到过小时候的趣事"

**不应该包含**:
- ❌ 一次性事件："用户昨天去了公园"
- ❌ 临时信息："用户明天要开会"（应该用日程工具）

### 3.3 应该存在Context的数据

**标准**: 临时状态 + 会过期 + 不持久化

**示例**:
- ✅ 当前时间、当前小时
- ✅ 临时位置（current_location）："现在在北京出差"
- ✅ 当前活动状态
- ✅ 临时情绪、当前任务

**特点**: 每次启动重新计算，不保存

## 四、数据流向

### 4.1 启动流程

```
1. 加载Profile (JSON) → UserProfile对象
2. UserProfile → State.user_profile (Dict)
3. UserProfile → 格式化为文本 → memory_context["user_profile"]
4. context_signals → 实时计算 → memory_context["realtime_context"]
5. memory_context → 注入System Prompt
```

### 4.2 运行时更新流程

#### 场景A: 提取到核心画像信息（如姓名）

```
用户输入 → extract_and_save_user_profile()
  ↓
提取到"用户姓名是张三"
  ↓
判断：是核心画像 → 更新Profile
  ↓
Profile.update_profile({"name": "张三"})
  ↓
保存到JSON文件
  ↓
同步到State.user_profile
  ↓
下次对话时自动加载
```

#### 场景B: 提取到次要偏好（如"喜欢去公园"）

```
用户输入 → extract_and_save_user_profile()
  ↓
提取到"用户喜欢去公园"
  ↓
判断：是次要偏好 → 保存到ROM
  ↓
save_user_memory("用户喜欢去公园", metadata={category: "preference"})
  ↓
保存到ChromaDB
  ↓
按需检索（Tool调用）
```

#### 场景C: 临时位置更新（如"现在在北京出差"）

```
用户输入 → extract_and_save_user_profile()
  ↓
提取到"用户现在在北京出差"
  ↓
判断：是临时位置 → 更新Context
  ↓
State.context_signals["current_location"] = "北京"
  ↓
不保存到Profile或ROM（临时信息）
  ↓
下次启动时重新计算
```

## 五、关键设计原则

### 5.1 分离原则

1. **Profile vs ROM**: 
   - Profile = 核心、结构化、快速访问
   - ROM = 次要、非结构化、按需检索

2. **Profile vs Context**:
   - Profile = 长期稳定、持久化
   - Context = 临时状态、不持久化

3. **ROM vs Context**:
   - ROM = 长期记忆、会保存
   - Context = 临时信息、不保存

### 5.2 同步原则

1. **Profile ↔ State.user_profile**: 双向同步
   - Profile更新 → 同步到State
   - State更新 → 同步到Profile

2. **Profile → memory_context**: 单向格式化
   - Profile → 格式化为文本 → memory_context

3. **ROM → memory_context**: 按需检索
   - Tool调用 → 检索ROM → 添加到memory_context

### 5.3 更新原则

1. **核心画像** → 更新Profile
2. **次要偏好** → 保存到ROM
3. **临时状态** → 更新Context（不保存）

## 七、总结

### 7.1 三层架构

1. **持久化层**: Profile (JSON) + ROM (ChromaDB)
2. **运行时状态层**: State.user_profile + State.context_signals + State.memory_context
3. **上下文层**: System Prompt

### 7.2 数据分类

- **Profile**: 长期稳定、结构化、核心画像
- **ROM**: 长期记忆、非结构化、次要信息
- **Context**: 临时状态、不持久化

### 7.3 数据流向

- **启动**: Profile → State → Context → System Prompt
- **运行时**: 提取 → 判断类型 → 更新对应层 → 同步
