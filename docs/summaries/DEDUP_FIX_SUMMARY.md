# 去重修复总结

**修复日期**: 2025-12-24  
**问题**: 语义去重未能识别"用户喜欢蓝色"和"用户最喜欢的颜色是蓝色"为重复  
**状态**: ✅ 已修复并测试通过

---

## 🐛 问题描述

在终端输出中发现，用户画像中存在明显的语义重复：

```xml
<user_profile>
- 用户喜欢蓝色
- 用户最喜欢的颜色是蓝色  <!-- 重复 -->
</user_profile>
```

原有的去重逻辑使用简单的 `split()` 分词，对中文不适用，导致无法识别语义重复。

---

## 🔍 根本原因

### 1. 关键词提取不适合中文

**原代码**:
```python
words1 = set(mem1.replace("用户", "").replace("的", "").split())
```

**问题**:
- `split()` 按空格分词，对中文无效
- 简单的 `replace()` 无法处理复杂的停用词
- 无法识别同义词（如"喜欢" vs "爱"）

### 2. 相似度阈值过高

- 原阈值: 0.7
- 对于中文短文本，阈值过高导致漏检

---

## ✅ 修复方案

### 1. 改进关键词提取函数

**新增特性**:

#### 1.1 文本标准化
```python
# 统一城市名称：移除"市"字
text_normalized = text
for city in cities:
    if f"{city}市" in text_normalized:
        text_normalized = text_normalized.replace(f"{city}市", city)
```

#### 1.2 实体识别
- **颜色词**: 红色、蓝色、绿色等 → 提取 `{颜色, 蓝色}`
- **城市名**: 上海、北京等 → 提取 `{上海, 地点信息}`
- **食物类型**: 火锅、日料等 → 提取 `{火锅, 食物喜好}`
- **音乐类型**: 轻音乐、古典等 → 提取 `{轻音乐, 音乐}`

#### 1.3 通用标签
为相似表达添加通用标签：
- "喜欢" / "爱" / "偏好" → `喜好表达`
- "所在地" / "住在" / "居住" → `居住表达`
- 颜色词 → `颜色`
- 地点词 → `地点信息`

#### 1.4 停用词过滤
```python
stopwords = {"用户", "的", "是", "在", "有", "和", "与", "或", "但", "而", 
             "最", "喜欢", "爱", "经常", "总是", "习惯", "偏好", "吃", "所", "住", "地"}
```

过滤无意义的组合词（如"用户所"、"用户住"）。

### 2. 降低相似度阈值

- 原阈值: 0.7
- 新阈值: 0.6

更敏感地检测中文短文本的语义重复。

### 3. 在 `nodes.py` 中对 `user_profile` 进行去重

`user_profile` 是字符串格式，需要先解析为列表：

```python
# 解析字符串为列表
profile_lines = [line.strip() for line in user_profile.split("\n") if line.strip()]

# 提取纯文本（移除 "- " 前缀）
profile_items = []
for line in profile_lines:
    if line.startswith("- "):
        profile_items.append(line[2:])

# 去重
deduplicated_items = context_manager.deduplicate_user_profile(profile_items)

# 重新格式化为字符串
user_profile_text = "\n".join([f"- {item}" for item in deduplicated_items])
```
