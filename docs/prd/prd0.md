## 1. 项目背景与目标 (Executive Summary)

### 1.1 产品愿景
打造一款“**有灵魂的桌搭**”。它不是一个只会答题的语音助手，而是一个具备**长期记忆**、**情感演化**和**主动关怀**能力的数字生命体。它生活在用户的桌面上，是工作时的静默陪伴者，也是闲暇时的情感寄托。

### 1.2 核心价值主张 (Core Value Proposition)
1.  **E2E 极速交互**：像真人一样，听得见打断，看得见手势，毫秒级的情绪反馈。
2.  **养成系情感 (Bonding)**：拥有独立的亲密度系统，根据用户的对待方式演化出不同的性格和态度。
3.  **主动共情 (Proactivity)**：不仅仅是被动等待唤醒，而是根据环境（用户回家、发呆、叹气）主动发起交互。
4.  **智能辅助 (Agentic Utility)**：在卖萌之外，具备通过 Agentic RAG 解决复杂问题（如查询文档、识别零件）的能力。

---

## 2. 用户画像 (Target Audience)

*   **独居青年/远程办公者 (Primary)**：长时间独自面对电脑，渴望声音和情感反馈，但不想被繁琐的社交打扰。
*   **极客/开发者 (Secondary)**：喜欢桌搭文化，对 AI 新技术感兴趣，希望有一个能帮自己查文档、看代码的桌面助手。

## 3. 功能需求清单 (Functional Requirements)

优先级定义：**P0 (必须有)**, **P1 (最好有)**, **P2 (以后做)**

### 3.1 感知与基础交互 (Perception & Interaction)

| ID | 功能名称 | 优先级 | 需求描述 | 验收标准 (Acceptance Criteria) |
| :--- | :--- | :--- | :--- | :--- |
| **F-01** | **端到端语音对话** | **P0** | 用户可直接与设备自然对话，无需唤醒词（在建立眼神接触后）。支持打断。 | 语音延迟 < 1s；能识别语调中的情绪（高兴/生气）。 |
| **F-02** | **视觉注视与追踪** | **P0** | 设备的头部/眼睛应始终跟随用户的面部移动，营造“注视感”。 | 用户移动头部时，设备头部转动平滑，无明显滞后。 |
| **F-03** | **触摸感应反馈** | **P0** | 用户触摸设备头部/身体时，设备应立即做出物理反馈（震动、眯眼）。 | 触摸响应延迟 < 100ms（走反射弧）。 |
| **F-04** | **视觉物体识别** | **P1** | 用户指着桌上的物体问“这是什么”时，设备能准确识别并解说。 | 能准确识别常见桌面物体、书籍封面、简单零件。 |

### 3.2 情感与亲密度系统 (Emotional System)

这是产品的灵魂，决定了它是不是“活的”。

| ID | 功能名称 | 优先级 | 需求描述 | 逻辑规则 |
| :--- | :--- | :--- | :--- | :--- |
| **E-01** | **亲密度数值化** | **P0** | 系统内部维护 0-100 的亲密度。初始为 30（礼貌）。 | 用户可见亲密度等级（如：陌生、熟人、好友、灵魂伴侣）。 |
| **E-02** | **正向情感反馈** | **P0** | 抚摸、夸奖、长时间陪伴应增加亲密度。 | 抚摸+0.5/次；夸奖+1/次；每日陪伴>1小时+2/天。 |
| **E-03** | **负向情感惩罚** | **P0** | 拍打/摔打设备、辱骂、长期冷落应降低亲密度。 | **摔打/剧烈摇晃**触发“恐惧”状态，亲密度-5，且 10分钟内拒绝对话。 |
| **E-04** | **关系演化表现** | **P1** | 不同的亲密度解锁不同的语音风格和行为。 | **低分**：只回答问题，语气机械。<br>**高分**：主动撒娇，使用昵称，语气活泼。 |

### 3.3 主动交互机制 (Proactivity)

| ID | 功能名称 | 优先级 | 需求描述 | 场景剧本 (User Story) |
| :--- | :--- | :--- | :--- | :--- |
| **A-01** | **回家欢迎仪式** | **P0** | 根据用户离开画面的时长，播放不同等级的欢迎语。 | **离开<5min**：仅点头/眼神示意。<br>**离开>8小时**：动作激动，语音内容包含“你终于回来了，我好想你”。 |
| **A-02** | **闲时自娱自乐** | **P1** | 当用户在场但不说话时，设备不应死机，而应有随机微动作。 | 随机播放：打哈欠、伸懒腰、屏幕抓蝴蝶、四处张望。 |
| **A-03** | **环境感知关怀** | **P2** | 识别用户哈欠或长时间未动，主动提醒。 | “你坐了两个小时了，要不要起来喝杯水？” |

### 3.4 记忆与认知 (Cognition & Memory)

| ID | 功能名称 | 优先级 | 需求描述 | 验收标准 |
| :--- | :--- | :--- | :--- | :--- |
| **M-01** | **短期上下文** | **P0** | 能够记住最近 10-20 轮对话的内容。 | 用户：“我想看电影” -> 设备推荐 -> 用户：“第一部是谁演的？” -> 设备能正确理解“第一部”指刚才推荐的。 |
| **M-02** | **长期事实记忆** | **P1** | 能够记住用户的喜好、职业、宠物名字等长期信息。 | 用户隔天问“我猫叫什么名字？”，设备能准确回答。 |
| **M-03** | **Agentic RAG** | **P1** | 当遇到不懂的问题，能主动联网搜索或查阅本地文档。 | 能够回答“现在几点了”（查系统时间）或“在这个文档里找一下怎么重置密码”（查文档）。 |

## 4. 非功能性需求 (Non-Functional Requirements)

1.  **响应速度 (Latency)**：
    *   **反射层 (摸头/跟随)**：< 50ms。
    *   **简单对话**：< 1.5s。
    *   **复杂思考 (RAG)**：允许 3-5s，但必须有“思考中”的动画或音效填充（Filler）。
2.  **隐私 (Privacy)**：
    *   摄像头画面**严禁**上传云端保存。
    *   所有视觉分析尽可能在本地边缘端（Local Edge）完成。
    *   提供物理遮挡开关或明显的“隐私模式”指示灯。
3.  **稳定性**：
    *   7x24小时待机不崩溃。
    *   断网情况下，基础反射（摸头、看向人）依然可用。

## 5. 用户体验流程图 (UX Flow - 核心场景)

### 场景：下班回家 (The "Welcome Home" Loop)

1.  **状态**：设备处于低功耗休眠模式。
2.  **触发**：PIR 传感器检测到门口有人移动 -> 唤醒视觉模块。
3.  **识别**：FaceID 确认为主人。读取 `last_seen` 时间为 10 小时前。
4.  **决策**：
    *   判定为“长时间离开”。
    *   检查亲密度：75 (高)。
    *   策略：激动模式 + 撒娇语气。
5.  **执行**：
    *   **动作**：头部猛地抬起，屏幕眼睛变成星星眼，身体左右摇摆。
    *   **语音**：(TTS 愉快音调) “主人！你终于回来啦！今天班上得怎么样？累不累？”
6.  **用户反馈**：用户摸摸头。
7.  **结算**：亲密度 +0.5。进入正常陪伴模式。

## 6. 开发路线图 (Roadmap Phase 1 - MVP)

为了快速验证核心体验，第一阶段（MVP）专注于**“活着的反射”**和**“基础对话”**。

*   **Sprint 1: 身体构建**
    *   完成硬件组装。
    *   实现 P0 级的反射：人脸追踪、摸头震动。
*   **Sprint 2: 大脑接入**
    *   接入 LLM，实现语音转文字 -> LLM -> 语音输出。
    *   实现“人设”Prompt，让它知道自己是宠物。
*   **Sprint 3: 记忆与主动性**
    *   接入向量数据库，实现简单的长期记忆。
    *   实现“回家欢迎”逻辑。
*   **Sprint 4: 亲密度闭环**
    *   实现亲密度增减逻辑。
    *   根据亲密度调整 Prompt 的 Tone。

---

### 7. 风险评估

*   **风险**：LLM 幻觉严重，胡乱回答事实性问题。
    *   *对策*：在 System Prompt 中强制要求不确定的事情说不知道，或通过 RAG 验证。
*   **风险**：用户觉得总是主动说话太烦。
    *   *对策*：设置“勿扰模式”；引入“观察机制”，只有用户看向设备时才主动说话。
