# Project Animus MVP 原型设计

## 设计理念说明

本文档包含3个不同设计理念的MVP概念原型图，每个原型聚焦于不同的用户体验重点。

---

## 原型1：极简主义设计 (Minimalist Design)

**设计理念**：聚焦核心交互，最小化视觉干扰，让用户专注于对话和情感连接。

**核心特点**：
- 简洁的圆形屏幕，仅显示眼睛动画
- 极简的物理形态，减少视觉噪音
- 强调触觉反馈和声音交互
- 适合专注工作场景

### ASCII 原型图

```
                    ┌─────────────────┐
                    │                 │
                    │    ╭─────╮     │
                    │   ╱   ●   ╲    │  ← 圆形屏幕（眼睛显示）
                    │  │   ●   │     │
                    │   ╲     ╱     │
                    │    ╰─────╯     │
                    └─────────────────┘
                         │     │
                    ┌────┘     └────┐
                    │               │
                    │   [触摸区域]   │  ← 触摸感应区
                    │               │
                    └───────────────┘
                         │     │
                    ┌────┘     └────┐
                    │               │
                    │   [底座]      │  ← 电机震动、灯光
                    │               │
                    └───────────────┘

【交互示意】
用户输入 "你好" → 屏幕眼睛眨眼 → 语音回复 "你好！"
用户按 'T' (模拟触摸) → 震动反馈 → 灯光变暖 → 语音 "喵~"
```

### 设计优势
- ✅ 视觉干扰最小，适合长时间陪伴
- ✅ 硬件成本低，易于实现
- ✅ 专注核心体验：对话和触摸

### 设计劣势
- ❌ 视觉表现力较弱
- ❌ 主动行为表现有限

## 原型2：情感化设计 (Emotional Design)

**设计理念**：强调情感表达和视觉反馈，通过丰富的动画和表情增强情感连接。

**核心特点**：
- 大尺寸屏幕，支持丰富的表情动画
- 可动的头部/身体结构
- 多彩灯光反馈系统
- 适合情感陪伴场景

用户输入 "我回来了" → 屏幕显示星星眼 → 头部左右摇摆 →

- ✅ 视觉表现力强，情感表达丰富
- ✅ 主动行为表现力强
- ✅ 用户情感连接更深

### 设计劣势
- ❌ 硬件成本较高
- ❌ 实现复杂度高
- ❌ 可能分散注意力

## 原型3：功能导向设计 (Functional Design)

**设计理念**：平衡实用性和情感体验，既满足智能助手功能，又保持情感陪伴。

**核心特点**：
- 中等尺寸屏幕，支持表情和文字显示
- 模块化设计，易于扩展
- 智能灯光系统（工作/休息模式）
- 适合多功能使用场景

### ASCII 原型图

```
                    ┌─────────────────────┐
                    │                     │
                    │   ┌───────────┐     │
                    │   │  (◕‿◕)    │     │  ← 表情显示区
                    │   │  ●     ●  │     │
                    │   └───────────┘     │
                    │                     │
                    │  ┌─────────────┐   │
                    │  │ [文字显示区] │   │  ← 文字/状态显示
                    │  └─────────────┘   │
                    │                     │
                    └─────────────────────┘
                         │         │
                    ┌────┴─────────┴────┐
                    │                   │
                    │  [触摸感应区]     │  ← 触摸反馈
                    │  [状态指示灯]     │  ← LED状态指示
                    │                   │
                    └───────────────────┘
                         │         │
                    ┌────┴─────────┴────┐
                    │                   │
                    │  [智能灯光环]     │  ← 可调色温/亮度
                    │  [电机震动模块]   │  ← 触觉反馈
                    │                   │
                    └───────────────────┘
                         │         │
                    ┌────┴─────────┴────┐
                    │                   │
                    │    [底座模块]     │
                    │  [扩展接口预留]   │  ← 未来扩展
                    │                   │
                    └───────────────────┘

【交互示意】
工作模式：用户说 "开始工作" → 屏幕显示专注表情 → 
灯光调至冷白光 → 语音 "好的，专注模式已开启"
休息模式：用户说 "我累了" → 屏幕显示关心表情 → 
灯光调至暖黄光 → 语音 "辛苦了，要不要休息一下？"
触摸交互：用户按 'T' → 震动反馈 → 灯光变暖 → 
屏幕显示开心表情 → 语音 "喵~"
```

### 设计优势
- ✅ 平衡实用性和情感体验
- ✅ 模块化设计，易于迭代
- ✅ 适合多种使用场景
- ✅ 扩展性强

### 设计劣势
- ❌ 设计复杂度中等
- ❌ 需要平衡功能优先级

## 原型对比总结

| 维度 | 原型1：极简主义 | 原型2：情感化 | 原型3：功能导向 |
|:---|:---|:---|:---|
| **视觉复杂度** | 低 | 高 | 中 |
| **硬件成本** | 低 | 高 | 中 |
| **实现难度** | 低 | 高 | 中 |
| **情感表达** | 中 | 高 | 中高 |
| **功能扩展性** | 低 | 中 | 高 |
| **适用场景** | 专注工作 | 情感陪伴 | 多功能使用 |
| **MVP适配度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |

---

## 推荐选择

**MVP阶段推荐：原型1（极简主义）或原型3（功能导向）**

**理由**：
- MVP阶段应聚焦核心价值验证，而非视觉表现
- 原型1实现成本最低，可快速验证"活着的反射"和"基础对话"
- 原型3在保持实用性的同时，为未来扩展预留空间

**后续迭代**：
- V2版本可考虑引入原型2的情感化元素
- 根据用户反馈调整设计方向
