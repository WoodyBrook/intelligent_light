# Agent 流程优化总结

**日期**: 2026-01-08  
**版本**: V2.1

## 优化目标

解决 Agent 流程中的两个核心问题：
1. **什么时候用条件判断 vs LLM判断？**
2. **如何避免重复的 LLM 调用？**

---

## 优化内容

### 1. 两阶段过滤逻辑（性能优化）

**问题**：之前的 `_is_simple_task` 直接调用 LLM 判断任务复杂度，导致不必要的延迟和成本。

**解决方案**：实现两阶段过滤

```
阶段1: 规则快速过滤（毫秒级，不调用LLM）
    ├── _is_clearly_simple() → 明确简单任务 → 直接返回 True
    ├── _is_clearly_complex() → 明确复杂任务 → 直接返回 False
    └── 不确定 → 进入阶段2

阶段2: LLM智能判断（仅对不确定场景）
    └── _llm_classify_task_complexity()
```

**改动文件**: `src/nodes.py`
- `_is_clearly_simple()`: 增强规则，识别问候语、控制命令、简单回复
- `_is_clearly_complex()`: 增强规则，识别条件逻辑、多步骤、明确查询词
- `_is_simple_task()`: 实现两阶段调用逻辑

### 2. 统一工具决策流程

**问题**：Plan Node 和 Reasoning Node 都在检测工具需求，导致重复 LLM 调用。

**解决方案**：Plan Node 统一决策，Reasoning Node 只执行

```
┌─────────────────────────────────────────────────────┐
│  Plan Node（决策中心）                               │
│  - 阶段1: 规则检测工具需求 (_rule_based_tool_detection)│
│  - 阶段2: LLM 检测（仅复杂任务）                      │
│  - 输出: required_tools 传递给 Reasoning Node        │
└─────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────┐
│  Reasoning Node（执行中心）                          │
│  - 复用 Plan Node 传递的 required_tools              │
│  - 不再重复调用 _detect_required_tools              │
│  - 只有 Plan 跳过时才做轻量级规则检测                 │
└─────────────────────────────────────────────────────┘
```

**改动文件**: `src/nodes.py`
- 新增 `_rule_based_tool_detection()`: 轻量级规则检测，不调用 LLM
- 修改 `plan_node()`: 两阶段工具检测（先规则后 LLM）
- 修改 `reasoning_node()`: 复用 Plan Node 的工具决策结果

### 3. 规则判断 vs LLM判断 清单

| 场景 | 方案 | 原因 |
|-----|------|------|
| 安全指令（停止、暂停） | ✅ 硬编码 | 必须即时响应 |
| 专注模式/冷却期检查 | ✅ 硬编码 | 业务规则，不需要智能 |
| 参数解析（亮度、色温） | ✅ 硬编码 | 正则更准确 |
| 反射响应（触摸、问候） | ✅ 硬编码 | 固定表达，不需要 LLM |
| 明确的查询词检测 | ✅ 规则优先 | 关键词匹配即可 |
| 指代消解（"那呢"） | 🤖 LLM | 需要理解上下文 |
| 不确定场景分类 | 🤖 LLM | 需要语义理解 |
| 复杂任务规划 | 🤖 LLM | 需要分解步骤 |
| 回复生成 | 🤖 LLM | 人设表达 |

## 性能提升

### 优化前
```
用户输入 → LLM判断简单/复杂 → Plan Node LLM检测工具 → Reasoning Node LLM检测工具
         (1次LLM调用)           (1次LLM调用)              (1次LLM调用)
         
最坏情况：3次 LLM 调用才能开始生成回复
```

### 优化后
```
用户输入 → 规则判断简单/复杂 → 规则检测工具 → 复用工具决策
         (毫秒级)              (毫秒级)        (0次额外调用)

明确场景：0次额外 LLM 调用
不确定场景：最多1次 LLM 调用（用于分类或工具检测）
```

### 预期收益
- **简单任务**（问候、控制命令）：减少 2-3 次 LLM 调用
- **明确查询**（天气、新闻）：减少 1-2 次 LLM 调用
- **复杂任务**（条件逻辑、多步骤）：保持原有流程，无退化
