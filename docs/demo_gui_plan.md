# V1 Demo GUI 方案

## 目标

创建一个轻量级GUI，用于现场实时演示V1功能，包括对话、亲密度系统、冲突管理、专注模式等。

## 技术选型

### 方案A：Streamlit（推荐）

- **优点**：快速开发、自动刷新、Web界面、易于展示数据可视化
- **缺点**：需要浏览器，稍微重一些
- **适用场景**：快速原型、数据展示、状态可视化

### 方案B：Tkinter

- **优点**：Python内置、轻量、原生窗口
- **缺点**：UI设计较繁琐、需要手动刷新
- **适用场景**：简单界面、不需要复杂交互

**推荐使用Streamlit**，因为：

1. 可以实时显示状态变化（自动刷新）
2. 易于展示亲密度进度条、状态卡片等
3. 可以同时显示对话历史和硬件状态
4. 开发速度快

### 2. 核心组件

#### 2.1 对话区域

- 显示对话历史（用户输入 + Animus回复）
- 实时滚动
- 区分用户和Animus的消息样式

#### 2.2 系统状态面板

- **亲密度显示**：进度条 + 数值 + 等级（stranger/acquaintance/friend/soulmate）
- **专注模式**：开关状态 + 开启原因
- **冲突状态**：显示当前冲突等级（L0-L3）+ 冷却倒计时
- **内部驱动**：无聊度、能量值

#### 2.3 硬件状态可视化

- **灯光**：
  - 颜色预览（根据color_temp显示）
  - 亮度滑块/进度条
  - 状态指示器（开/关）
- **电机**：
  - 震动类型显示（purr/stutter/none）
  - 强度指示器

#### 2.4 传感器模拟

- 按钮：触摸、摇晃
- 点击后触发sensor事件

### 3. 实现细节

#### 3.1 文件结构

```
Neko_light/
├── demo/
│   ├── gui_demo.py          # Streamlit主程序
│   ├── state_visualizer.py  # 状态可视化组件
│   └── hardware_simulator.py # 硬件状态模拟器
└── demo_runner.py            # 启动脚本（可选）
```

#### 3.2 与现有系统集成

**方案1：直接调用（推荐）**

- GUI作为前端，直接调用`main.py`中的工作流
- 使用`event_manager`发送事件
- 从`state_manager`读取状态

**方案2：WebSocket/API桥接**

- 主程序运行在后台
- GUI通过API获取状态
- 更解耦，但实现复杂

**推荐方案1**，因为：

- 实现简单
- 实时性好
- 不需要额外网络层

#### 3.3 状态同步机制

```python
# 在gui_demo.py中
import streamlit as st
from state_manager import StateManager
from event_manager import EventManager
from graph import build_graph

# 初始化
state_manager = StateManager()
event_manager = EventManager()
app = build_graph()

# 在循环中
current_state = state_manager.get_state_summary(state)
# 更新GUI显示
```

#### 3.4 硬件状态可视化

使用Streamlit的组件：

- `st.progress()` - 亮度进度条
- `st.color_picker()` - 颜色预览（只读）
- `st.metric()` - 数值显示
- `st.status()` - 状态指示器

### 5. 实现步骤

1. **创建Streamlit基础框架**（1-2小时）
   - 布局设计
   - 输入框和按钮
   - 基础状态显示

2. **集成状态管理器**（1小时）
   - 连接`state_manager`
   - 实时读取状态
   - 更新显示

3. **实现硬件可视化**（1-2小时）
   - 灯光状态可视化
   - 电机状态显示
   - 颜色预览

4. **添加传感器模拟**（30分钟）
   - 触摸按钮
   - 摇晃按钮
   - 事件触发

5. **创建演示脚本**（1小时）
   - 预设场景
   - 一键演示
   - 状态重置

6. **优化和美化**（1-2小时）
   - UI美化
   - 动画效果
   - 响应式布局

## 替代方案：命令行增强版

如果时间紧张，也可以先做一个**命令行增强版**：

- 使用`rich`库美化输出
- 彩色状态显示
- 表格展示系统状态
- 进度条显示亲密度

**优点**：开发快（1-2小时）

**缺点**：不如GUI直观

## 推荐方案

**优先使用Streamlit GUI**，因为：

1. 开发时间合理（5-8小时）
2. 可视化效果好
3. 适合现场演示
4. 易于扩展

如果时间紧急，可以先做命令行增强版，后续再升级到GUI。
