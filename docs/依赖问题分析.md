## 问题总结

### 根本原因
项目在 `requirements.txt` 中列出了所有依赖，但：
1. 用户可能没有安装所有依赖
2. 某些依赖是可选的（如 MCP、feedparser），不应该阻止系统启动
3. 直接导入会导致模块加载时立即失败

### 修复策略
采用**可选导入模式**：
```python
try:
    import optional_module
    MODULE_AVAILABLE = True
except ImportError:
    optional_module = None
    MODULE_AVAILABLE = False
    logging.warning("⚠️ 模块未安装，相关功能将不可用")
```

### 已修复的文件
1. ✅ `src/mcp_manager.py` - MCP 和 OAuth 可选导入
2. ✅ `src/content_providers.py` - feedparser 可选导入

### 建议
1. **开发环境**：运行 `pip install -r requirements.txt` 安装所有依赖
2. **生产环境**：根据实际需求选择性安装依赖
3. **文档说明**：在 README 中说明哪些依赖是可选的
