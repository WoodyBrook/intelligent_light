# 情绪识别与记忆机制

本文档描述 Neko_light 项目中关于用户情绪的识别、反馈处理以及基于情感的记忆存储机制。

---

## 概述

系统通过多层机制来感知和记忆用户的情绪状态：

1.  **实时情绪识别**：基于关键词匹配和 LLM 推理，实时判断用户当前对话的情绪（如开心、疲惫、焦虑）。
2.  **偏好情感分析**：在特定场景（如音乐推荐、新闻推送）下，分析用户对内容的反馈情感（正向/负向）。
3.  **情感记忆存储**：将识别到的情感倾向作为元数据存储到向量数据库 (ChromaDB)，用于未来的个性化服务。
4.  **亲密度系统**：基于交互的情感质量动态调整亲密度，影响系统的语气和行为。

---

## 涉及文件

| 文件 | 功能 |
| :--- | :--- |
| [content_providers.py] | 提供基于关键词的轻量级情绪识别算法，用于内容推荐 |
| [memory_manager.py] | 负责从对话中提取偏好并分析情感（Sentiment Analysis），存储到数据库 |
| [nodes.py] | 推理节点，整合 LLM 的情感分析能力，计算亲密度变化 |
| [conflict_handler.py] | 处理冲突状态和道歉逻辑，涉及负面情绪的缓解机制 |

## 实时情绪识别机制

### 1. 基于关键词的快速识别 (Rule-based)

在无需调用 LLM 的场景（如快速音乐推荐）下，系统使用关键词匹配来推断用户情绪。

*   **实现位置**: `src/content_providers.py` -> `MusicRecommender.get_mood_from_conversation`
*   **支持情绪**:
    *   `happy`: 开心, 高兴, 快乐, 哈哈, 😊
    *   `sad`: 难过, 伤心, 郁闷, 😢
    *   `tired`: 累, 疲惫, 困, 😴
    *   `energetic`: 兴奋, 激动, 精神, 💪
    *   `anxious`: 焦虑, 紧张, 担心, 😰
    *   `focused`: 工作, 学习, 专注, 忙

### 2. 基于 LLM 的深度推理 (LLM-based)

在主推理流程中，LLM 负责分析用户的深层意图和情绪状态。

*   **实现位置**: `src/nodes.py` -> `reasoning_node`
*   **机制**: System Prompt 中包含 `意图分析思考` 指引，要求 LLM 分析 "用户可能处于什么情绪或状态"，并据此生成 `voice_content` 和调整个性化语气。

## 偏好情感分析与记忆 (Sentiment Memory)

系统不仅记忆"发生了什么"，还记忆用户"对此的感觉"。这是通过 `MemoryManager` 中的提取器实现的。

### 1. 音乐偏好情感提取

*   **方法**: `extract_music_preference`
*   **逻辑**:
    1.  检测对话中是否包含评价类关键词。
    2.  **判定情感**：
        *   `positive`: 喜欢, 好听, 不错, 很棒, 推荐
        *   `negative`: 不喜欢, 讨厌, 难听, 太吵, 换一首
        *   *注：优先检测负面情感，防止误判（如"好的，我知道你不喜欢"被误判为正面）。*
    3.  **生成记忆**:
        ```python
        {
            "content": "用户不喜欢周杰伦的音乐",
            "category": "music",
            "sentiment": "negative",  # 关键元数据
            "confidence": 0.8
        }
        ```

### 2. 新闻/话题兴趣提取

*   **方法**: `extract_news_preference`
*   **逻辑**: 使用正则表达式匹配明确的意图表达（如 `r"不想看.*新闻"` vs `r"多推.*新闻"`）。

### 3. 存储结构 (ChromaDB Metadata)

所有包含情感的记忆在存入 Vector DB 时，都会附带 `sentiment` 元数据：

```json
{
    "page_content": "用户不喜欢周杰伦的音乐",
    "metadata": {
        "category": "music",
        "sentiment": "negative",
        "timestamp": 1705234567,
        "importance": 5
    }
}
```

> [!TIP]
> 这种存储结构允许系统在检索时进行过滤，例如：在推荐时排除 `sentiment="negative"` 的内容，或优先检索 `sentiment="positive"` 的高愉悦感记忆。

## 亲密度与冲突处理 (Intimacy & Conflict)

情绪识别直接影响系统的内部状态机：

1.  **夸奖识别**: `reasoning_node` 会检测用户的夸奖（"你真棒"），增加 `intimacy_level`。
2.  **冲突检测**: `conflict_handler` 检测用户的攻击性语言或负面情绪，触发 "L1~L3" 冲突状态，导致系统进入冷却期或拒绝执行某些命令。
3.  **道歉识别**: 在冷却期内，系统会识别用户的道歉意图，从而提前结束冷却，恢复正常交互。
